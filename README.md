# Advanced App Tracker

Десктопный трекер активности приложений на **PyQt6** с функциями самоконтроля. Отслеживает, в каком приложении вы работаете, ведёт статистику по дням и неделям, делит приложения на продуктивные/непродуктивные, позволяет ставить лимиты времени и — при желании — **мешает себя закрыть** (пароль на выход + сторож, перезапускающий приложение, если его «убить»).

## Возможности

- **Отслеживание активного окна** (Windows / macOS / Linux-X11, с fallback по заголовку окна) и запись длительностей в SQLite.
- **Статистика**: суммарное время за сегодня и неделю, разбивка по продуктивности, история за произвольный период. Текущая активная сессия показывается в интерфейсе сразу, не дожидаясь переключения приложения.
- **Графики**: круговая диаграмма распределения (matplotlib) и тренд дневного использования (pyqtgraph).
- **Лимиты** дневные/недельные с предупреждениями при приближении и опциональным завершением приложения при превышении.
- **Детекция простоя** — время бездействия не засчитывается.
- **Системный трей**: сворачивание вместо закрытия, запуск свёрнутым.
- **Telegram-бот**: по команде `/start` отправляет администратору сводку за сегодня и неделю, топ приложений и лимиты; в сообщении есть кнопка обновления.
- **Самоконтроль (опционально)**:
  - пароль на выход из интерфейса;
  - **сторож (guardian)** — фоновый процесс, перезапускающий приложение, если его завершить (например, через Диспетчер задач). Корректный выход через меню сторож не трогает; есть защита от «шторма» перезапусков.
- **Автозапуск** при старте Windows.
- Тёмная тема.

## Установка

Нужен **Python 3.10+**.

```bash
pip install -r requirements.txt
```

`aiogram` нужен для Telegram-бота. Если `pynput`, `bcrypt` или `pygetwindow` недоступны, приложение запускается с деградацией соответствующих функций: детекция простоя, bcrypt-хеширование пароля или fallback определения активного окна могут быть отключены.

## Запуск

```bash
python -m app_tracker
# или
python run.py
```

На Windows `run.py` также настраивает совместимую политику event loop для
Telegram-бота. Этот лаунчер используется автозапуском и сторожем.

## Telegram-бот

1. Создайте бота через [@BotFather](https://t.me/BotFather) и получите token.
2. Откройте `Настройки -> Telegram`.
3. Вставьте `Telegram Bot Token`.
4. Укажите `Admin IDs` — числовые Telegram user id через запятую, пробел или точку с запятой.
5. Сохраните настройки. Бот запускается автоматически; перезапуск приложения не требуется.

Команда `/start` доступна только указанным администраторам. Если пользователь не в списке, бот ответит его id, чтобы его можно было добавить в настройки.

## Разработка

Проект сейчас запускается напрямую из исходников. Установите runtime-зависимости:

```bash
pip install -r requirements.txt
```

Быстрая проверка импортов и синтаксиса:

```bash
python -m compileall app_tracker run.py
```

`pyproject.toml`, тесты и вспомогательная генерация иконки удалены из текущего состава проекта, поэтому команды `pip install -e .[dev]`, `ruff` и `pytest` в репозитории больше не настроены.

## Данные и логи

Хранятся в каталоге пользователя:

- Windows: `%LOCALAPPDATA%\AppTracker\`
- macOS: `~/Library/Application Support/AppTracker/`
- Linux: `~/.local/share/AppTracker/`

Там находятся `app_tracker.db`, `app_tracker.log` и служебные файлы сторожа. Настройки Telegram-бота и пароль на выход также хранятся в SQLite.

## Заметки по безопасности

- Пароль на выход защищает **только выход через интерфейс**, а не завершение процесса через системные средства. Для последнего предназначен сторож.
- Сторож перезапускает приложение при неожиданном завершении, но **всегда** уважает корректный выход (через меню/пароль) и прекращает попытки при слишком частых перезапусках, чтобы не зациклиться.
- Хеширование пароля использует `bcrypt`, а при его отсутствии — salted SHA-256.
- Telegram-бот отдаёт статистику только id из списка `Admin IDs`. Сам token хранится в локальной базе настроек, поэтому не коммитьте пользовательскую БД и логи.
